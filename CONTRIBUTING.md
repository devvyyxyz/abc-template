# Contributing

Thanks for your interest in contributing to this template. Contributions can include bug reports, feature requests, documentation improvements, or pull requests that add useful scaffolding.

How to contribute

1. Fork the repository and create a branch for your change: `git checkout -b my-feature`.
2. Make small, focused commits with clear messages.
3. Run `make install-scripts` to ensure helper scripts are executable locally if needed.
4. Open a pull request describing the change and why it's useful.

Issue guidelines

- Use the issue templates when filing bugs or feature requests.
- Provide reproduction steps, Minecraft version, and any relevant logs or screenshots.

Pull request checklist

- [ ] My changes are small and focused.
- [ ] I updated documentation where appropriate (README, data/ namespace README, etc.).
- [ ] I did not commit secrets or built artifacts.

Code style

This project is a collection of text files and scripts for Minecraft packs. Keep scripts POSIX-compatible where possible and document any non-obvious commands.

Thank you for improving the template!
